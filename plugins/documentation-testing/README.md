# Documentation Testing

Doc testing, example validation, link checking

## What's Included

### Agents
- **Doc Tester** - Specialized agent for Doc testing, example validation, link checking

### Commands
- `/test-docs` - Quick-access command for documentation-testing workflows

### Skills
- **Doc Testing Patterns** - Pattern library and knowledge base for documentation-testing

## Quick Start

1. Copy this plugin to your Claude Code plugins directory
2. Use the agent for guided, multi-step workflows
3. Use the command for quick, targeted operations
4. Reference the skill for patterns and best practices

## Usage Examples

```
# Use the command directly
/test-docs analyze

# Use the command with specific input
/test-docs generate --context "your project"

# Reference patterns from the skill
"Apply doc-testing-patterns patterns to this implementation"
```

## Key Patterns

- Follow established conventions for documentation-testing
- Validate inputs before processing
- Document decisions and rationale
- Test outputs against requirements
- Iterate based on feedback
