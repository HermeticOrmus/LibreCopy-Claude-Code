# Intermediate — docs at scale + testing

## Docs in CI

- Spec lint (Spectral for OpenAPI, vale for prose)
- Link checking (broken internal + external links)
- Doctest (verify code examples run)
- Diff against last version (breaking-change detection)
- Render preview per PR

## Information architecture

- Diátaxis: per-page categorization
- Search-first design: users search, don't browse
- Cross-linking: explicit, not implicit
- Versioning: archive old versions, default to current

## Style guide

- Tone: friendly + precise; not patronizing, not stiff
- Person: second-person for tutorials ("you"); third-person for reference ("the user")
- Verb tense: present
- Voice: active
- Vocabulary: ubiquitous (each term defined once, used consistently)
