# Beginner — technical writing mindset

## Mindset shifts

1. **Documentation is a product, not paperwork.** Users include future-you. Treat docs the way you'd treat the product.
2. **Audience determines structure.** Newcomer ≠ experienced ≠ lookup-only. Don't mix.
3. **Diátaxis sorts the confusion.** Tutorial / How-to / Reference / Explanation. Pick one per page.
4. **Examples that work beat prose that explains.** Code that runs > description of code.
5. **Docs-as-code or docs-rot.** In repo, in PR, in CI — or stale within months.

## Your first doc

Pick an API or feature you understand. Use `/api-docs` (or `/readme-engineering` in v0.3) to design the IA.
